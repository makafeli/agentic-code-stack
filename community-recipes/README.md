# Community Recipes

A directory of community-submitted stack variants, tool configurations, and workflow setups. Anyone can contribute via pull request.

## How to contribute

1. Create a subdirectory for your recipe (e.g., `community-recipes/ios-dev/` or `community-recipes/data-engineering/`)
2. Include a README.md explaining what your setup does, which tools it includes, and why you chose them
3. Submit a pull request

## What makes a good recipe

- It solves a specific use case or workflow
- It's documented -- someone else should be able to understand and use it
- It works -- you've actually used this setup
- It includes any config files, AGENTS.md variants, or setup instructions needed

## Review bar

Light review: is it documented? Does it work? Not a formal RFC. The best recipes may be promoted into the official pipeline/rails docs with contributor credit.

## Discussions

Share your setup and get feedback in [GitHub Discussions](https://github.com/.../discussions) (Show Your Stack category) before submitting a PR if you'd like community input first.
