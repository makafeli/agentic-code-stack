Layer 01

Documentation & Code Context

Feed the agent the right facts. Shrink what gets read.
Ref MCP
ref-tools · MCP
Smart-chunked API docs. Returns ~200 relevant tokens instead of an 80k page.
MCP tools (auto-invoked)
ref_search_documentation(query) ref_read_url(url) search_web(query)
No slash command. Agent calls these when docs are needed.
Context7
Upstash · MCP + CLI
Version-specific docs from source repos. Triggered by use context7.
In prompt
use context7 use library /supabase/supabase
Slash command
/context7:docs <query>
CLI
ctx7 setup --claude ctx7 library <name> <query> ctx7 docs <libraryId> <query>
Pitlane MCP
eresende · MCP
Tree-sitter index of your repo. Retrieves only the symbols the agent asked for.
MCP tools
ensure_project_ready investigate(query) locate_code(symbol) search_content(snippet) search_files(pattern) trace_path(from, to) read_code_unit(id) find_usages(symbol) watch_project

Layer 03

Memory & Task Tracking

Survive context loss. Cross-session continuity.
Beads (bd)
gastownhall · CLI + Plugin
Dependency-aware issue graph in your repo. bd ready finds unblocked work.
CLI — daily flow
bd prime bd ready bd show <id> bd update <id> --claim bd close <id> --reason "..." bd create "title" -t feature -p 1 bd remember "insight" bd dep add <id> <parent> --type discovered-from
Plugin slash commands
/beads:ready /beads:create /beads:workflow @task-agent
OpenSpec
Fission-AI · Skills
Specs as durable memory. Context doesn't die when the chat ends.
Slash commands
/opsx:propose "<idea>" /opsx:apply /opsx:archive /opsx:verify
Specs live in openspec/specs/ — readable by every future agent session.
