Layer 01

Documentation & Code Context

Feed the agent the right facts. Shrink what gets read.
Ref MCP
ref-tools · MCP
Smart-chunked API docs. Returns ~200 relevant tokens instead of an 80k page.
MCP tools (auto-invoked)
ref_search_documentation(query) ref_read_url(url) search_web(query)
No slash command. Agent calls these when docs are needed.
Context7
Upstash · MCP + CLI
Version-specific docs from source repos. Triggered by use context7.
In prompt
use context7 use library /supabase/supabase
Slash command
/context7:docs <query>
CLI
ctx7 setup --claude ctx7 library <name> <query> ctx7 docs <libraryId> <query>
Pitlane MCP
eresende · MCP
Tree-sitter index of your repo. Retrieves only the symbols the agent asked for.
MCP tools
ensure_project_ready investigate(query) locate_code(symbol) search_content(snippet) search_files(pattern) trace_path(from, to) read_code_unit(id) find_usages(symbol) watch_project
Layer 02

Token & Output Efficiency

Less context, less waste, longer sessions.

Natural language

graphify
/graphify · Skill
Codebase → knowledge graph in graphify-out/. Query the graph instead of re-reading files.
Commands
/graphify [path|url] graphify query "<question>" graphify explain "<node>" /graphify --update
If graphify-out/graph.json exists, treat codebase questions as graph queries.

Layer 03

Memory & Task Tracking

Survive context loss. Cross-session continuity.
GitHub Issues
gh CLI + MCP
Native to any GitHub repo. Reference #<n> in commits to auto-close. The portable default.
CLI
gh issue list gh issue create --title "..." gh issue view <n> gh issue close <n> --comment "..." gh issue develop <n> --checkout
MCP (github)
list_issues · issue_read · issue_write
Linear
MCP + Printing Press CLI
For teams on Linear. Compound SQL via the Printing Press linear CLI (local mirror, ~50ms). Use ENG-123 IDs in branches/PRs.
Setup
npx -y @mvanhorn/printing-press install linear
Or connect the Linear MCP for read/write issues, cycles, projects.

Agent Interfaces

Printing Press
printingpress.dev
Generate agent-native Go CLIs from API specs, HAR files, or URLs. One command outputs Go binary + Claude Code skill + OpenClaw skill + MCP server. Every CLI ships with local SQLite mirror, compound queries, and structured output.
Setup
go install github.com/mvanhorn/cli-printing-press/v4/cmd/printing-press@latest npx -y @mvanhorn/printing-press install starter-pack git clone https://github.com/mvanhorn/cli-printing-press.git
